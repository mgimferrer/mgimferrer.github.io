## Publications section

### General rules
- **30 publications** total (no corrigenda/errata included).
- **Numbered 1 (oldest, 2016) → 30 (newest, 2026).**
- **Displayed newest-first** in the HTML (p30 at top, p1 at bottom).
- Number badges use `data-num="N"` on the `<li>` and `content: attr(data-num)` in CSS — no JS counter.
- Each `<li>` also carries `data-doi="10.xxxx/..."` (lowercase) used by `publications.js` to inject TOC images.
- Citation format: **ACS style** — `Authors. Title. *J. Abbrev.* **Year**, *vol.* (issue), pages. DOI`
- In HTML: `<em>` for journal, `<strong>` for year, `<em>` for volume — per ACS convention.
- **Martí Gimferrer's name** is always `<strong>Gimferrer, M.</strong>` (bold).
- **Corresponding author** marker `<sup>*</sup>` appears only on **p22, p23, p25, p30**.

### Layout & CSS classes
Each `<li>` carries two classes:
- `pub-even` (data-num is even) or `pub-odd` (data-num is odd)
- Even → left border in `--forest-soft`; odd → left border in `--gold`

When `publications.js` injects a TOC image it wraps the `<p class="pub-ref">` in a `<div class="pub-row">`:
- **Odd-numbered** pubs: image on the **left** (`flex-direction: row`)
- **Even-numbered** pubs: image on the **right** (`flex-direction: row-reverse`)
- On mobile (≤ 760 px) all rows stack vertically regardless of parity.

TOC images use class `pub-toc-img` (300 px wide, max-height 240 px).

### TOC image pipeline

TOC images are no longer stored manually. The pipeline is:

1. **Download:** run `scripts/fetch-tocs.py` from repo root (re-runnable; skips already-downloaded files).
   ```
   pip install requests cloudscraper Pillow
   python3 scripts/fetch-tocs.py
   ```
2. **Output:** images land in `assets/tocs/` and a manifest is written to `data/tocs.json` (maps DOI → filename).
3. **Render:** `publications.js` loads `data/tocs.json` at page load, then for each `<li data-doi>`:
   - Uses the local `assets/tocs/<filename>` if the DOI is in the manifest.
   - Falls back to a Crossref API image link if available.
   - Renders without a thumbnail if nothing is found — never shows a broken image.

**Publisher coverage of fetch-tocs.py:**

| Prefix | Publisher | Strategy | Notes |
|--------|-----------|----------|-------|
| 10.1039/ | RSC | direct | confirmed working |
| 10.1021/ | ACS | cloudscraper | confirmed working |
| 10.1002/ | Wiley | cloudscraper | confirmed working |
| 10.1016/ | Elsevier | cloudscraper | validated by aspect ratio |
| 10.3762/ | Beilstein | direct | confirmed working |
| 10.1063/ | AIP | — skipped | 403 even with bypass |
| 10.3390/ | MDPI | — unknown | not in publisher routing |

DOIs for p23 and p21 are AIP (10.1063/) — no TOC image can be fetched automatically.

### Adding a new publication
1. Increment the total count in `<p class="section-intro">` (currently "30 publications").
2. Also update the count in the About section and hero `<p class="lede">`.
3. Add `<li class="pub-even|pub-odd" data-num="N+1" data-doi="10.xxxx/...">` at the **top** of `<ol class="pub-list">`.
4. Re-run `scripts/fetch-tocs.py` to pick up the new TOC automatically.
